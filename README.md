# riscv-cpu

\# Based on FSM-Fused Architecture: 16-Instruction RISC-V CPU Design



\## 项目简介

本项目独立研发了一款支持 RV32I 核心指令集（共 16 条核心指令）的 32 位低功耗处理器微架构。区别于传统的流水线设计，本项目采用了分层有限状态机融合架构，将复杂的执行控制流高效融合于核心模块中，并引入前瞻控制进行时序优化。项目已全流程跑通 VCS 和 Verdi 前仿验证与 Design Compiler 逻辑综合，成功实现 200MHz 时序收敛。



\## 核心架构与技术特性



\### 1. 微架构设计

\* \*\*指令集支持\*\*：完全实现 16 条核心通用指令，涵盖算术逻辑（add, sub, and, or, sll, srl, addi, ori）、访存（lw, sw）及分支跳转（beq, bne, jal, jalr）等核心拓扑。

\* \*\*控制单元\*\*：采用精心优化的分层有限状态机代替传统流水线线性的阶段划分，在融合模块内实现高效的动态调度，大幅优化面积开销。

\* \*\*前瞻控制优化\*\*：通过状态跳转的前瞻性预测控制，将指令平均执行周期由 5 缩减至 3。



\### 2. 前端验证与仿真

\* \*\*工具链\*\*：Synopsys VCS 和 Verdi

\* \*\*验证方案\*\*：搭建分层 Testbench，利用官方测试集（HEX 机器码流式加载）开展全指令集边界测试，利用 Verdi 实施精确的波形 Debug 与时序定位。



\### 3. 逻辑综合

\* \*\*工具链\*\*：Synopsys Design Compiler

\* \*\*工艺节点\*\*：基于标准 65nm 工艺库及 SRAM Macro 进行了物理级综合约束。

\* \*\*综合策略\*\*：将 SRAM 设为 dont\_touch 避免 Macro 破坏；应用 compile\_ultra -no\_autoungroup 策略保持模块层级，为后端布局布线提供最优解。

\* \*\*时序成果\*\*：在 0.8 的时序裕量因子严苛约束下，成功消除 Setup 和 Hold 违例，达成 200MHz 时序收敛。



\## 项目文件树

\* \*\*rtl/\*\* : 核心硬件设计源码，包括 riscv.v, ControlUnit.v, ALU.v, PC.v, RF.v 等独立模块

\* \*\*includes/\*\* : 全局参数与宏定义头文件

\* \*\*tb/\*\* : 用于全指令集仿真的 Testbench 平台

\* \*\*sim/\*\* : 基于 VCS 的仿真运行环境及自动化 Makefile

\* \*\*syn/\*\* : 基于 Design Compiler 的逻辑综合约束脚本 syn.tcl
