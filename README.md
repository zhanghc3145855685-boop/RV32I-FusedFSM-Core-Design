# RV32I-FusedFSM-Core-Design

## 项目概述
本项目旨在设计并验证一款高性能、轻量级的 32 位 RISC-V 处理核心。该架构支持 RV32I 核心指令集，通过创新的**分层有限状态机（FSM）融合架构**，规避了传统流水线设计在短指令集下的流水线冒险与开销问题，实现了低功耗与高执行效率的平衡。

项目完成了从 RTL 设计、仿真验证到后端逻辑综合的全流程闭合。在 65nm 工艺节点下，该核心在 Design Compiler 中成功通过时序约束，达成 200MHz 目标频率，验证了其在嵌入式控制场景下的工业落地潜力。

---

## 核心架构与工程亮点

### 1. 微架构优化
* **指令集覆盖**：完整支持 16 条核心通用指令，涵盖 ALU 算术逻辑、访存操作及控制流跳转。
* **分层FSM融合控制**：摒弃冗余的流水线级联，采用状态机融合设计，在单一执行通路内实现指令的快速调度，有效缩减逻辑面积。
* **性能指标**：通过前瞻性控制逻辑优化，将指令平均执行周期（CPI）由 5 优化至 3。

### 2. 设计验证
* **工具链**：Synopsys VCS / Verdi
* **方法学**：构建全指令集自动化仿真平台。通过机器码流式加载进行边界测试，利用 Verdi 进行时序精细化 Debug，实现功能逻辑与设计意图的完全对齐。

### 3. 逻辑综合与物理实现
* **工艺节点**：基于标准 65nm 工艺库与 SRAM Macro 进行综合约束。
* **综合策略**：采用 `compile_ultra` 高级综合策略；针对 SRAM 定义 `dont_touch` 属性保护 Macro 边界；通过层级保持策略优化，为后端布局布线（APR）预留充足设计裕量。
* **时序目标**：在严苛约束下消除 Setup/Hold 时序违例，成功收敛至 200MHz。

## 工程实现背景

本项目所有的核心 RTL 源码、仿真验证环境及综合约束脚本，均系**基于离线开发环境下的全自主设计与重构**。

由于受限于研发环境的网络隔离及严苛的权限管控，本项目在无原始工程备份的情况下，完全依靠设计规格（Design Spec）与逻辑验证分析，完成了代码的复现、手工调试与语法合规性闭合。该过程充分体现了核心模块的逻辑重构能力、对工具链（VCS/DC）的深度掌控力以及在限制性约束下的复杂工程协作能力。
